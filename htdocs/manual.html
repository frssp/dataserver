사용 안내

사내 Zotero 서버 사용법입니다 — 계정을 만들고, 브라우저에서 라이브러리를 보고, Zotero 데스크톱 앱을 동기화하는 방법을 안내합니다.

메타데이터 전용 동기화. 이 서버는 참고문헌 항목·컬렉션·태그·노트·저장된 검색만 동기화합니다. 첨부 파일이나 PDF는 동기화하지 않습니다. 파일은 추가한 그 기기에만 남습니다.
⚠️ 동기화 전에 백업하세요. 이 서버로 처음 동기화하기 전에, 기존 라이브러리를 Zotero RDF로 내보내 백업해 두는 것을 강력히 권장합니다. 방법은 아래 3. 데스크톱 앱 동기화를 참고하세요.

0 Zotero가 처음이라면

Zotero를 처음 쓴다면, 이 사내 서버를 설정하기 전에 Zotero 자체의 기본 사용법을 먼저 익히는 것을 권합니다. 아래 두 가지면 충분합니다.

  • 공식 사이트 둘러보기 — zotero.org에서 Zotero가 무엇이고 무엇을 할 수 있는지 살펴보세요.
  • 입문 영상 보기 — 짧은 소개 영상과 강의 재생목록으로 전체 사용 흐름을 익히세요.

입문 영상 (YouTube) → 강의 재생목록 (YouTube) →

기본기를 익혔다면, 아래 1. 계정 만들기부터 이 사내 서버 설정을 시작하세요.

1 계정 만들기

어디서든 로그인하려면 먼저 사용자명과 비밀번호가 필요합니다. 이 서버에 접속할 수 있는 사람은 누구나 직접 가입할 수 있습니다.

  1. 가입 페이지(Sign Up)에서 사용자명·이메일·비밀번호를 입력합니다.
  2. 제출하면 개인 라이브러리와 API 키가 자동으로 만들어집니다.
  3. 가입 직후 웹 라이브러리(Web Library)에 바로 로그인됩니다.
입력 규칙: 사용자명은 3~32자(영문·숫자·. _ -), 비밀번호는 8자 이상, 이메일은 올바른 형식이어야 합니다.

계정 만들기 →

이미 가입했다면 같은 사용자명과 비밀번호로 웹 라이브러리에 로그인하면 됩니다.

2 웹 라이브러리에서 보기

웹 라이브러리는 브라우저에서 참고문헌을 보고 내보내는 화면입니다. 설치가 필요 없습니다.

웹 라이브러리 열기

할 수 있는 것

  • 3분할 화면 — 왼쪽 컬렉션, 가운데 항목 목록, 오른쪽 항목 상세.
  • 라이브러리 전환 — 내 개인 라이브러리와 내가 속한 그룹 라이브러리.
  • 필터·검색 — 태그로 좁히거나 제목·저자·연도로 검색.
  • 정렬·열 선택 — 열 머리글 클릭으로 정렬, 표시할 열 선택(브라우저에 저장됨).
  • 항목 상세 — 선택한 항목의 정보(Info)·노트(Notes)·태그(Tags) 탭.
  • 내보내기 — 항목이나 컬렉션을 우클릭해 BibTeX·BibLaTeX·RIS·CSL JSON·CSV로 내보내기.

로그인 / 로그아웃

  • 로그인할 때 ID 칸에는 사용자명 또는 이메일 중 아무거나 사용할 수 있습니다.
  • 계정을 바꾸려면 로그아웃 후 다시 로그인하세요. (인증 정보는 이 브라우저에만 저장됩니다.)
웹 라이브러리는 보기·내보내기 용도입니다. 참고문헌을 추가·편집하려면 이 서버와 동기화되는 Zotero 데스크톱 앱(아래 3번)을 사용하세요.

3 데스크톱 앱 동기화

Zotero 데스크톱 앱이 zotero.org 대신 이 서버와 동기화하도록 설정합니다. 한 번만 하면 됩니다. Zotero 7 이상에서 동작하며, 가급적 최신 버전을 권장합니다(브라우저 로그인은 최신 버전 기준 — 구버전은 아래 로그인 단계의 API 키 방식 사용).

동기화 전에 백업하세요. 서버를 바꾸거나 처음 동기화하기 전에, 기존 라이브러리를 내보내 백업해 두는 것을 권합니다. Zotero에서 파일 File → 라이브러리 내보내기 Export Library… → 형식으로 Zotero RDF를 선택해 저장하세요. 문제가 생겨도 그 파일로 복원할 수 있습니다.

  3. 아래 두 항목을 설정합니다. (검색창에 이름을 붙여넣고, 더블클릭해서 값을 변경)

변경 후 Zotero를 재시작합니다.

2로그인

  1. 설정 Settings → 동기화 Sync 를 엽니다.
  2. 로그인(Sign In / Link Account) 을 클릭하면 브라우저에 로그인 페이지가 열립니다.
  3. 가입할 때 쓴 사용자명(또는 이메일)과 비밀번호를 입력하고 확인합니다. 데스크톱 앱이 접근 권한을 자동으로 받아옵니다.
  4. 다시 동기화 Sync 화면에 계정이 연결되어 표시됩니다. 도구 모음의 동기화 버튼(초록색 회전 화살표)을 눌러 동기화합니다.
브라우저 로그인이 안 열리면(또는 구버전이면): Account 페이지에서 API 키를 발급한 뒤, Zotero의 동기화 설정에 그 키를 직접 입력해도 됩니다.

평소 사용

  • 데스크톱 앱에서 평소처럼 참고문헌을 추가·편집·정리합니다.
  • 동기화 버튼을 누르면(또는 자동 동기화로) 변경 내용이 서버에 반영됩니다.
  • 다른 컴퓨터에서도 같은 설정을 하고 같은 계정으로 로그인하면 동일한 라이브러리를 사용할 수 있습니다.
파일은 동기화되지 않습니다. 첨부 파일·PDF는 업로드되지 않으니, 동기화 설정의 "파일 동기화(File Syncing)" 옵션은 꺼 두세요. 이 서버는 라이브러리(메타데이터) 동기화만 동작합니다.

4 그룹 라이브러리

그룹 라이브러리는 여러 사람이 하나의 참고문헌 모음을 함께 쓰는 공간입니다.

  • 내가 속한 그룹은 자동으로 나타납니다 — 데스크톱 앱 왼쪽 창과 웹 라이브러리의 라이브러리 전환 목록에 표시됩니다.
  • 그룹 라이브러리를 편집할 수 있는지는 내 역할(멤버/관리자/소유자)과 그룹 설정에 따라 다릅니다.
  • 그룹에 추가되려면 그룹 소유자나 서버 관리자에게 요청하세요.

내가 속한 그룹은 Account 페이지에서 확인할 수 있습니다.

그룹 만들기

Account 페이지의 Groups 탭에서 직접 그룹을 만들 수 있습니다.

  1. 그룹 이름을 입력합니다.
  2. 공개 범위를 선택합니다 — Public(누구나 온라인에서 열람) 또는 Private Membership(초대된 멤버만 열람·참여).
  3. 라이브러리 편집 권한을 정합니다 — 모든 멤버(All Members) 또는 관리자만(Admins Only).
  4. 필요하면 설명을 적고 Create Group을 누릅니다.
  5. 생성 후 그룹의 멤버 관리에서 다른 사용자를 추가하면, 그 사람의 라이브러리에도 이 그룹이 나타납니다.
그룹을 만든 사람이 소유자가 됩니다. 멤버 추가·역할 변경·그룹 삭제도 같은 Groups 탭에서 할 수 있습니다.

5 API 키 고급

API 키는 스크립트나 외부 도구가 Zotero Web API로 라이브러리를 읽고 쓸 수 있게 해줍니다. 데스크톱 앱과 웹 라이브러리 로그인은 키를 자동으로 처리하므로 평소엔 신경 쓸 필요가 없습니다.

  • Account 페이지의 API Keys에서 키를 보거나 새로 만들 수 있습니다.
  • 요청마다 Zotero-API-Key 헤더로 키를 보냅니다. 예: GET /users/<id>/items
  • 키는 비밀번호처럼 다루세요. 더 이상 쓰지 않는 키는 삭제합니다.

자주 묻는 질문

다른 컴퓨터에서 PDF가 안 보여요.

정상입니다. 이 서버는 메타데이터만 동기화합니다. 참고문헌 항목은 동기화되지만, 첨부 파일은 추가한 컴퓨터에만 남습니다.

설정 직후 동기화가 실패해요.

extensions.zotero.api.url이 위 예시 주소와 같은지, streaming.enabled가 false인지, Zotero를 재시작했는지 확인하세요. 그리고 Zotero가 로그인에 쓰는 브라우저에서 이 사이트가 열리는지 확인하세요.

클라이언트에서 로그인을 누르면 localhost 페이지가 열려요.

서버 설정 문제입니다. 클라이언트가 여는 로그인 주소는 서버가 알려주는 값(ZOTERO_BASE_URL)이라서, 이 값이 기본값 http://localhost:8080/로 남아 있으면 localhost가 열립니다. 관리자가 docker/.env의 ZOTERO_BASE_URL을 클라이언트에서 접속 가능한 실제 주소(끝에 / 포함, 예: http://서버IP:8080/)로 바꾼 뒤 설정을 재생성해야 합니다. (아래 관리자용 참고)

로그인할 때 사용자명을 써요, 이메일을 써요?

둘 다 됩니다. 가입 때 등록한 사용자명이나 이메일 중 무엇으로도 로그인할 수 있습니다.

비밀번호를 잊어버렸어요.

스스로 재설정하는 기능은 없습니다. 관리자에게 새 비밀번호 설정을 요청하세요.

zotero.org 공식 계정을 그대로 쓸 수 있나요?

아니요. 이 서버는 별도의 사내 서버입니다. 여기서 새로 가입하세요. zotero.org와는 독립적입니다.

관리자용

관리자는 Admin 패널(슈퍼유저 인증으로 보호됨)에서 서버를 관리합니다.

  • 사용자 추가·조회·삭제, 사용자 비밀번호 재설정
  • 그룹 라이브러리 생성·설정, 멤버 추가·삭제
  • 임의 사용자의 API 키 발급·삭제
  • 서버 상태 확인(사용자·그룹·항목 수, 스키마 버전)

자체 가입(Sign Up)이 기본적으로 열려 있어, 사용자는 관리자 없이도 가입 페이지에서 직접 계정을 만들 수 있습니다.

로그인 주소 설정 (ZOTERO_BASE_URL)

클라이언트가 로그인 시 여는 주소는 서버의 ZOTERO_BASE_URL에서 나옵니다. 기본값(http://localhost:8080/)이면 다른 PC의 클라이언트가 자기 localhost를 열어 실패합니다. docker/.env에서 클라이언트가 접속 가능한 실제 주소(끝 슬래시 포함)로 바꾼 뒤 설정을 재생성하세요:

# docker/.env ZOTERO_BASE_URL=http://<서버주소>:8080/ # 적용 (docker/ 디렉터리에서) docker compose up -d php-fpm docker compose exec php-fpm rm -f include/config/config.inc.php docker compose restart php-fpm

확인: docker compose exec php-fpm grep BASE_URI include/config/config.inc.php 결과가 실제 주소로 나오면 됩니다.

User Guide

How to use this self-hosted Zotero server — create an account, browse your library in the browser, and sync the Zotero desktop app.

Metadata-only sync. This server syncs your references, collections, tags, notes, and saved searches. It does not sync attached files or PDFs — those stay on the device where you added them.
⚠️ Back up before you sync. Before your first sync to this server, we strongly recommend exporting your existing library as a Zotero RDF backup. See 3. Sync the desktop app for how.

0 New to Zotero?

If you've never used Zotero before, it's worth learning the basics of Zotero itself before setting up this server. These two resources are enough to get started:

  • Explore the official site — see what Zotero is and what it does at zotero.org.
  • Watch a quick intro — a short video and a playlist walk through the overall workflow.

Intro video (YouTube) → Video playlist (YouTube) →

Once you're comfortable with the basics, start the server setup at 1. Create an account below.

1 Create an account

You need a username and password before you can sign in anywhere. Anyone with access to this server can self-register.

  1. Open the sign-up page and fill in a username, email, and password.
  2. Submit — your personal library and an API key are created automatically.
  3. You'll be signed straight into the Web Library.
Rules: username 3–32 chars (letters, digits, . _ -), password at least 8 characters, and a valid email address.

Create your account →

Already registered? Just sign in to the Web Library with the same username and password.

2 Browse in the Web Library

The Web Library is a read-and-browse view of your references in the browser — no install required.

Open Web Library

What you can do

  • Three-panel layout — collections on the left, the item list in the middle, item details on the right.
  • Switch libraries — your personal library and any group libraries you belong to.
  • Filter & search — narrow by tag, or search title / creator / year.
  • Sort & choose columns — click a column header to sort; pick which columns are visible (saved in your browser).
  • Item details — Info, Notes, and Tags tabs for the selected item.
  • Export — right-click an item or collection to export to BibTeX, BibLaTeX, RIS, CSL JSON, or CSV.

Sign in / sign out

  • You can sign in with either your username or your email.
  • To switch accounts, sign out and sign back in. (Your credentials are stored only in this browser.)
The Web Library is for viewing and exporting. To add and edit references, use the Zotero desktop app (next section), which syncs with this server.

3 Sync the desktop app

Point the Zotero desktop app at this server so it syncs here instead of zotero.org. This is a one-time setup. Works on Zotero 7 or newer; the latest version is recommended (browser sign-in needs a recent version — older ones can use the API-key method in Step B).

Back up before you sync. Before switching servers or syncing for the first time, export your existing library as a backup — in Zotero: File → Export Library… → choose Zotero RDF. If anything goes wrong, you can restore from that file.

APoint Zotero at this server

  1. In Zotero, open Edit → Settings (on macOS: Zotero → Settings).
  2. Go to the Advanced tab → Config Editor, and accept the warning.
  3. Set these two preferences (use the search box to find each, double-click to edit):
PreferenceValue
extensions.zotero.api.url… ← trailing / recommended
extensions.zotero.streaming.enabledfalse
Trailing slash recommended. Add the / at the end to match the example. (Zotero 7 usually works without it, but matching the example exactly is safest.)

Restart Zotero after changing these.

BSign in

  1. Open Settings → Sync.
  2. Click Sign In / Link Account. Zotero opens a login page in your browser.
  3. Enter the username (or email) and password you registered with, and confirm. The desktop app picks up your access automatically.
  4. Back in Sync, your account shows as linked. Click the Sync button (green circular arrows) in the toolbar to sync.
If the browser sign-in doesn't appear (or on older versions): create an API key on the Account page and paste it directly into Zotero's sync settings.

Day-to-day

  • Add, edit, and organize references in the desktop app as usual.
  • Press the Sync button (or let auto-sync run) to push changes to the server.
  • On another computer, repeat this setup and sign in with the same account to get the same library.
Files don't sync. Attachments and PDFs are not uploaded. Keep the "File Syncing" options in Sync settings turned off — only library/metadata sync works here.

4 Group libraries

Group libraries let several people share one collection of references.

  • Groups you're a member of appear automatically — in the desktop app's left pane and in the Web Library's library switcher.
  • Whether you can edit a group's library depends on your role (member / admin / owner) and the group's settings.
  • To be added to a group, ask the group owner or an administrator.

You can review the groups you belong to on your Account page.

Create a group

You can create a group yourself from the Groups tab on your Account page.

  1. Enter a group name.
  2. Choose the membership type — Public (anyone can view it online) or Private Membership (only invited members can view and join).
  3. Set library editing — All Members or Admins Only.
  4. Add an optional description, then click Create Group.
  5. After it's created, add other users from the group's member management; the group then appears in their libraries too.
Whoever creates the group becomes its owner. You can add members, change roles, and delete the group from the same Groups tab.

5 API keys advanced

An API key lets scripts or third-party tools read and write your library through the Zotero Web API. You normally don't need this — the desktop app and Web Library handle keys for you.

  • View and create keys on your Account page, under API Keys.
  • Send the key with each request as a Zotero-API-Key header, e.g. GET /users/<id>/items.
  • Treat keys like passwords. Delete any key you no longer use.

FAQ

My PDFs aren't showing up on another computer.

That's expected — this server syncs metadata only. The reference entry syncs, but the attached file stays on the computer where you added it.

Sync fails right after setup.

Double-check extensions.zotero.api.url matches the address above, that streaming.enabled is false, and that you restarted Zotero. Make sure you can open this site in the same browser Zotero uses for sign-in.

Clicking sign-in in the client opens a localhost page.

That's a server-side setting. The login URL the client opens comes from the server (ZOTERO_BASE_URL); if it's left at the default http://localhost:8080/, the client opens localhost. An administrator needs to set ZOTERO_BASE_URL in docker/.env to the real address clients can reach (with a trailing /, e.g. http://server-ip:8080/) and regenerate the config. (See the admin section below.)

Do I sign in with my username or email?

Either works. You can sign in with the username or the email you registered with.

I forgot my password.

There's no self-service reset. Ask an administrator to set a new password for you.

Can I use my official zotero.org account?

No. This is a separate, self-hosted server. Register an account here; it's independent of zotero.org.

For administrators

Administrators manage the server from the Admin panel (protected by the super-user credentials), where you can:

  • Add, inspect, and delete users; reset user passwords.
  • Create and configure group libraries, and add/remove members.
  • Issue or revoke API keys for any user.
  • Check server status (user / group / item counts, schema version).

Self-registration is open by default, so end users can create their own accounts via the Sign Up page without admin involvement.

Login address (ZOTERO_BASE_URL)

The URL the client opens at sign-in comes from the server's ZOTERO_BASE_URL. If it's left at the default (http://localhost:8080/), clients on other machines open their own localhost and fail. Set it in docker/.env to the real address clients can reach (with a trailing slash) and regenerate the config:

# docker/.env ZOTERO_BASE_URL=http://<server-address>:8080/ # apply (from the docker/ directory) docker compose up -d php-fpm docker compose exec php-fpm rm -f include/config/config.inc.php docker compose restart php-fpm

Verify: docker compose exec php-fpm grep BASE_URI include/config/config.inc.php should show the real address.
